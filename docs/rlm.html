Mode guide

gjc rlm research mode

Run a notebook-style research session with a persistent Python kernel, live notebook capture, and a synthesized report — without giving the session write/edit tools.

Start a research session

gjc rlm                                    # open an interactive research session
gjc rlm "What drives the spike in orders?"  # seed the session with a question
gjc rlm --data ./datasets/DATA.md           # load a project/data description

Use gjc rlm when the task is exploration: analyze data, inspect files, search the web, compute statistics, make plots, and produce a report. Use a normal gjc session when the task is implementation or repository mutation.

What tools does RLM get?

RLM is intentionally constrained. It exposes a hard-gated research toolset rather than the full coding-agent mutation surface.

  • python — persistent Python kernel for analysis, plotting, and computation.
  • read — inspect files, docs, images metadata, archives, URLs, and data sources.
  • web_search — gather current external information when needed.
  • Inspection-only bash prefixes, goal, and complete_research support session control and completion.
Boundary: RLM does not expose normal repository mutation tools such as edit, write, task, or browser automation. The allowlist is asserted after tool-registry assembly so leaked tools fail the launch.

Artifacts

Each RLM run writes durable research artifacts under .gjc/rlm/<session>/:

  • notebook.ipynb — every executed Python cell, written live with atomic temp-rename updates and post-write validation.
  • report.md — synthesized when the session exits.
  • metadata and state files used to resume/report the research session.

Recommended loop

# 1. Create a small data brief if useful
cat DATA.md

# 2. Start research
gjc rlm --data DATA.md "Find the strongest signal and explain the evidence"

# 3. Let the agent compute/search/read, then ask it to complete the report
# 4. Review .gjc/rlm/<session>/report.md and notebook.ipynb
