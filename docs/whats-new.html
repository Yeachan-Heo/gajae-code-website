What's new in v0.7.9

Web search you can trust over any endpoint, broader notifications, and Intel macOS binaries back in release CI — plus a round of reliability fixes.

🔎 Web search you can trust over any endpoint

Native web search now works through proxies and custom OpenAI-compatible endpoints, recovers real citations even when a provider omits structured annotations, and fails closed instead of inventing sources.

  • Native search over proxies. In auto mode, when a model is served through a proxy/custom endpoint and canonical native search credentials are absent, the active model's own credential + baseUrl drive native search, dispatched by wire protocol — Anthropic (anthropic-messages), OpenAI-compatible (openai-responses/completions), or Gemini (Generative Language) — each falling back to DuckDuckGo when the endpoint can't search.
  • Inline citation recovery. When a search demonstrably ran but the provider returned no structured url_citation annotations, sources are harvested from inline markdown links and bare URLs. Recovery is gated on a real-search signal, so a stray prose URL in a non-search answer is never promoted to a citation.
  • Codex stays on official OpenAI. The ChatGPT-backed codex native search is selected only for genuine OpenAI/ChatGPT endpoints; custom/proxy OpenAI-compatible models reuse their own credentials instead of local Codex OAuth.
  • Fail-closed Anthropic. Anthropic falls back to DuckDuckGo when Claude answers from stable knowledge without searching, and on web_search error blocks.
  • Opt-in insane read fallback. A keyless, opt-in fallback can recover blocked public URLs for read — no TLS impersonation, credential storage, or auto-installed dependencies.

📲 Notifications: Discord, Slack & Telegram attachments

The configure-once notifications SDK keeps its single action-needed/reply contract — and now reaches more clients with richer payloads.

  • Discord and Slack adapters. Both join Telegram on the same transport-agnostic contract, so action-needed signals and replies can route to those clients without an upstream change.
  • Telegram attachments. The Telegram daemon forwards agent images and accepts user-sent photos and files, for inbound and outbound attachments.
  • Threaded Mode verification. gjc notify setup verifies Telegram Threaded Mode and falls back to a flat private chat when topics are unavailable.
Off unless configured. No endpoint exists until notifications are enabled and a token is present. The Telegram daemon is a reference client of the generic SDK, not the upstream topology. Follow the Telegram onboarding guide →

🧰 Tooling & platform

  • Intel macOS binaries are back. Release CI builds standalone darwin-x64 binaries again on the supported macos-15-intel runner, so Intel Mac users get prebuilt binaries instead of npm/Bun or source-only installs.
  • C# LSP. Built-in detection for csharp-ls, with omnisharp preserved as a fallback when csharp-ls is unavailable.
  • tmux session preservation. GJC-managed tmux sessions survive attach/disconnect instead of being torn down, and launch no longer implicitly attaches.
  • Truncated tool calls rejected. Incomplete streamed tool calls fail fast instead of executing against a mismatched schema with partial arguments.
