Gajae Remote

A high-level design for a thin phone steering wheel that observes and nudges PC-side Gajae Code sessions without becoming a remote shell.

Thin steering wheel

Gajae Remote is the product direction for checking session status, reading bounded summaries, and submitting short prompts from a phone while the actual coding authority stays on the PC.

Authority boundary

Not a shell: a mobile client should not expose raw tool results, environment variables, diffs, secrets, system prompts, or host command execution. It should show the smallest useful session view.
  • PC-side owner remains the source of truth.
  • Phone clients get summarized state and explicit submit endpoints.
  • Pairing and scopes should be narrow, revocable, and auditable.

Useful states

Idle, working, blocked, disconnected, and rejected-submit states are more useful on mobile than raw transcript streaming. The remote view should tell the operator what action is safe and what action is unavailable.
